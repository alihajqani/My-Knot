# Type Safety

## üìå The Concept
**Type Safety** prevents you from performing illogical operations, like adding a "Text" to a "Number".

*   **Python (Dynamic):** You won't know there is an error until the code actually runs and crashes (Runtime Error).
*   **TypeScript (Static):** The code will not compile if there is an error. You catch bugs *before* running the code (Compile-time Error).

## üíª Code Comparison

### Python (Unsafe)
```python
def add_money(current, deposit):
    return current + deposit

# ‚ö†Ô∏è DANGER: This runs, but crashes later!
# One is a number, one is a string.
print(add_money(100, "50")) 
# Crash: TypeError
```

### TypeScript (Safe)
```TypeScript
function addMoney(current: number, deposit: number): number {
    return current + deposit;
}

// ‚úÖ SAFE: The editor highlights this line in RED.
// You cannot even run this code.
console.log(addMoney(100, "50")); 
// Error: Argument of type 'string' is not assignable to parameter of type 'number'.
```
